Рейтинг

Просто старая обрезанная вёрстка

  • Следуй спецификации HTML 4.0 или 4.1
    <!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.0 Transitional//EN">
  • Указывай кодировку
    <meta http-equiv="Content-Type" content="text/html; charset=utf-8">
  • Но не используй Unicode-символов, которых нет в windows-1251
  • Основной инструмент - таблицы. Проблемные таблицы...

Только допустимые теги и атрибуты

Лечим теги, Айболит

  • Таблицам следует обнулять все отступы
    <table cellpadding="0" cellspacing="0" style="margin: 0; padding: 0; border-collapse: collapse;">
  • Оставляем пустым title для Android
  • Оборачиваем письмо в <div id="mailsub"> от отступов в Mail.ru
  • Инлайним все стили! Подключаемые файлы стилей плохо работают
  • Использовать как можно меньше divов (в outlook'е лишние отступы)

Картинки любят внимание

  • Для важных картинок - только тег img, без areamap
  • Для неважных в фоне - атрибут background (не в outlook*)

  • * только background-image для body или так, так или так

       без background-position и без background-repeat

  • Обязательно указывать как минимум ширину width: 100%
  • Обязательно обнулять границы картинки border: 0
  • Желательно обнулять высоту строки картинки line-height: 0
  • Между img и закрытием ячейки таблицы не должно быть пробелов и переносов строк <img src"..."></td>

Нужно больше золота внимания

  • Нужно хранить картинки рассылок на своих серверах. Навсегда
  • В связи с этим нужно указывать абсолютные пути
  • Чтобы заменить картинку, используй новый путь. Они кэшируются
  • Нельзя использовать png. Только jpg и gif
  • Учитывайте, что gif не анимируется в outlook - остаётся первый кадр
  • Можно кодировать в base64, но нужно следить за весом письма
  • Часто приходится нарезать картинки на части

Оформляй

Стили

  • Фиксированная максимальная ширина - от 600px до 800px
  • Полные HEX-цвета, не капс. НЕТ: color: #DDD, ДА: color: #dddddd
  • Никакого абсолютного позиционирования
  • Без составных правил - только простые стили
  • Никаких тегов списков, имитируйте их символом • (&#8226;)
  • Обращение к классам - через атрибут class, а не через точку [class="block"] {...}, а не .block {...}

Стили

  • Никаких псевдо-элементов (:before, :after и т.п.)
  • Никаких составных селекторов (.title + .body)
  • Медиа-выражения уже работают в gmail (с 2016)
  • Скрытие display: none работает в gmail (с 2016)
  • Не надо делать горизонтальные отступы ячейкой таблицы с символом пробела &nbsp; с размером шрифта или gif'кой 5px
    Смело можно использовать padding (кроме тегов <p>, <div> и <a>)

Адаптивность писем

  • Desktop first
  • Медиа-выражения: не min-width, а max-device-width
  • Float'инг колонок
  • Подмена фона со скрытием img
  • label должны быть всегда сверху

Адаптивность писем

  • Обязательно применяй text-size-adjust: 100% для iPhone
  • Обязательно отключай зум
    <meta name="viewport" content="width=device-width, user-scalable=no" />
  • Адаптивность картинок: max-width: 100%; height: auto
  • Retina-ready картинки - @2x
  • Используй соответствующие типы инпутов: tel, email, number, date, datetime, search, url

Динамика в письмах

Измененяй состояния при наведении и фокусе, без JS

.content {
  display: none;
}
.title:hover + .content {
  display: block;
}

Думай о пользователе

  • Укажи target="_blank" ссылкам

  • Оставляй отступы вокруг контента не менее 10px, чтобы контент не заезжал на границы

UTM-метки

  • Источник кампании - utm_source.

    пример: newsletter

  • Канал кампании - utm_medium

    пример: email

  • Название кампании - utm_campaign

    пример: cats-lecture

  • Ключевое слово - utm_term

    пример: cats

  • Содержание кампании - utm_content

    пример: header
